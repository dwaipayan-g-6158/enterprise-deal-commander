# Improvement Proposals

Nine standalone feature-proposal documents. Each describes a *proposed* design — **not**
implementation status. Cross-check [`../../roadmap.md`](../../roadmap.md#improvement-proposals)
and the code for what has actually shipped.

| Proposal | What it proposes |
|---|---|
| [Closed-Lost Autopsy](./Closed-Lost%20Autopsy.md) | A loss-intelligence module — structured causal-chain capture, cross-loss pattern detection, timeline reconstruction, and proactive loss-risk scoring. Thesis: losses cluster by cause and follow detectable patterns. |
| [Dashboard widget suggestions](./Dashboard%20widget%20suggestions.md) | The dashboard as a "decision surface" — every widget answers one of six questions (money at stake, what's broken, where stuck, what next, is the forecast real, what changed). Seven non-negotiable ship-blocker widgets. |
| [Deal Memory Module](./Deal%20Memory%20Module.md) | A living institutional knowledge system capturing and surfacing intelligence from every closed deal to fight institutional amnesia and compress seller ramp. |
| [Deal Roster suggestions](./deal%20roster%20suggestions.md) | The complete Roster page UX — a triage table with toolbar, saved-view tabs, filters, bulk actions, inline preview, and status bar for one person triaging 10–50 deals. |
| [Risk Engine](./Risk%20Engine.md) | The "Risk Engine v2.0" redesign — replace binary pattern detection with a continuous 0–100 score across seven independent risk dimensions that is contextual, trend-aware, ranked, and explainable in under 30 seconds. |
| [Risk Engine2](./Risk%20Engine2.md) | Reconciles the dimensional engine with the named patterns as one unified three-layer system (dimensions = vital signs, patterns = diagnoses); a firing pattern amplifies its dimension. |
| [Portfolio Risk Analysis](./Portfolio%20Risk%20Analysis.md) | Detect correlated risk across team members and product lines — the "correlated risk blind spot" — elevating EDC from deal-level to portfolio-level intelligence. |
| [Pipeline Analytics](./Pipeline%20Analytics.md) | Turn EDC into a pipeline intelligence engine — flow dynamics, forecast reliability, segmentation, temporal patterns, and bottleneck analytics. |
| [Settings Engine Tuning, Automation, and Integrations](./Settings%20Engine%20Tuning,%20Automation,%20and%20Integrations%20Improvements.md) | The control panel governing 200+ tunable parameters across the analytical modules, plus automation rules and integrations — making configuration accessible, safe, and auditable. |
